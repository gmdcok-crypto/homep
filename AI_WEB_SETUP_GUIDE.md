# 🤖 AI 웹 자동화 컨트롤(제어) 필수 초기 세팅 가이드

이 문서는 AI 코딩 에이전트(예: Antigravity)가 여러분의 컴퓨터에서 웹 브라우저를 직접 띄우고, 화면을 보면서 자동화 테스트 및 에러를 수정할 수 있도록 환경을 구성하는 완벽한 가이드입니다. 

---

## 1. 브라우저 컨트롤 패키지 설치 (Playwright / Puppeteer)

AI가 화면을 보고 제어하려면 사람을 대신해 클릭하고 캡처를 할 "투명 인간 알바생(브라우저 제어 라이브러리)" 설정이 필요합니다. 개발 환경에 맞게 두 가지 중 하나를 선택해 주세요. (최근에는 Playwright를 강력히 권장합니다.)

### 옵션 A : Playwright (모든 브라우저 전천후 뷔페형 - 권장!)
Playwright는 크롬뿐만 아니라 파이어폭스, 아이폰 사파리까지 구동 테스트를 다 해볼 수 있는 마이크로소프트의 최신 만능 도구입니다. 이 도구는 **"조종기를 설치(npm)한 뒤, 내가 실제로 쓸 브라우저 엔진만 따로 다운로드(npx)"** 하는 2단계 방식을 사용합니다.

1. 터미널을 열고, 브라우저 제어 **조종기 껍데기**를 먼저 내 프로젝트에 설치합니다. (마이크로소프트 개발 도구이므로 `npx playwright install` 명령어 기반을 제공받기 위함입니다.)
   ```bash
   npm install -D playwright
   ```
   * *💡 참고: 명령어의 `-D`는 '완성된 앱을 실제 배포할 때는 제외하고, 내 컴퓨터에서 개발(수리)할 때 연장통으로만 쓰겠다'는 똑똑한 옵션입니다.*

2. 조종기가 설치되었으면, **"크롬 코어 엔진(가장 필수적임)" 하나만 인터넷에서 딱 1번 다운로드**해 줍니다!
   ```bash
   npx playwright install chromium
   ```
   * *💡 왜 npx 인가요?: 굳이 내 프로젝트 공간에 무거운 브라우저 파일을 설치하지 않고, 일회성(실행)으로 다운로드만 쫙 받고 빠지는 것이 효율적이기 때문입니다.*

### 옵션 B : Puppeteer (1세대 구글 크롬 전용 일체형 햄버거 세트)
Puppeteer는 오로지 구글 크롬(Chrome)만을 조작하기 위해 만들어진 초기 모델입니다. "어차피 크롬만 조종할 거니까 한 번에 다 다운받게 하자!"라는 철학으로 일체형 세트로 만들어져 있어 명령어가 한 줄입니다.

* 터미널에 아래 명령어 1줄만 치면 조종기와 무거운 크롬 브라우저 엔진이 알아서 폴더 안에 전부 다운로드(설치)됩니다.
   ```bash
   npm install -D puppeteer
   ```

## 2. 권한 탈취 방지를 위한 API 키 보안 세팅 (.env)

브라우저 컨트롤 설정이 끝났다면, AI(예: 구글 제미나이 등)가 똑똑한 두뇌를 쓸 수 있도록 권한(API Key)을 부여해야 합니다. 하지만 **이 키를 코드에 그냥 노출하면 남들이 복사해 가는 대형 보안 사고**가 터집니다. 그래서 우리는 안전 금고인 `.env` 파일을 만들어야 합니다.

### API 키값 작성 3단계

1. **파일 생성:** 
   프로젝트의 가장 바깥 폴더(루트 경로)에 새 파일을 하나 만들고, 이름을 반드시 **`.env`** 로 지정합니다. (앞에 점(.)이 필수입니다.)

2. **키 입력:**
   만든 빈 `.env` 파일 안에, 발급받은 Google AI Studio (Gemini) API 키를 따옴표나 띄어쓰기 없이 아래처럼 딱 붙여 넣습니다.

   ```env
   GEMINI_API_KEY=AIzaSyAxxxxxx(여기에_실제_발급받은_키값_붙여넣기)
   ```
   * *💡 만약 React + Vite 환경에서 웹 프론트엔드 코드 내부 자바스크립트가 직접 이 키를 읽어야 한다면 이름을 `VITE_GEMINI_API_KEY=` 로 이름을 바꿔 넣으시면 됩니다.*

3. **파일 저장 (.gitignore 확인):**
   작성을 마쳤다면 파일을 꼭 저장(Ctrl+S)해 주세요! (일반적으로 React/Vite로 프로젝트를 처음 만들었다면, 해커가 훔쳐가지 못하게 `.env` 파일은 인터넷(GitHub)에 안 올라가도록 세팅되어 있습니다.)

## 3. 웹서버 실행 (마무리)

완벽하게 AI의 눈과 손(브라우저 컨트롤), 그리고 지능(API 키) 세팅이 완료되었습니다! 

터미널에 아래 명령어를 칩니다.
```bash
npm run dev
```

터미널에 로컬 주소(예: `http://localhost:5173`)가 파랗게 뜨면 성공입니다. 이제 AI에게 "방금 띄운 저 화면 좀 고쳐줘!" 라고 명령해 보세요. 알아서 척척 처리합니다!
